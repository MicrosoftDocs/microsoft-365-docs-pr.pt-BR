<span data-ttu-id="52743-106">Como o tráfego geral da Internet pode ser arriscado, as redes de organizações típicas reforçam a segurança com dispositivos de borda, como servidores proxy, Interrupção e Inspeção de SSL, dispositivos de inspeção de pacotes e sistemas de prevenção de perda de dados.</span><span class="sxs-lookup"><span data-stu-id="52743-106">Because general Internet traffic can be risky, typical organization networks enforce security with edge devices such as proxy servers, SSL Break and Inspect, packet inspection devices, and data loss prevention systems.</span></span> <span data-ttu-id="52743-107">Leia sobre alguns dos problemas com dispositivos de interceptação de rede em [Usando dispositivos de rede de terceiros ou soluções no tráfego do Microsoft 365](https://support.microsoft.com/help/2690045/using-third-party-network-devices-or-solutions-with-office-365).</span><span class="sxs-lookup"><span data-stu-id="52743-107">Read about some of the issues with network interception devices at [Using third-party network devices or solutions on Microsoft 365 traffic](https://support.microsoft.com/help/2690045/using-third-party-network-devices-or-solutions-with-office-365).</span></span>

<span data-ttu-id="52743-p102">No entanto, os endereços IP e nomes de domínio DNS usados pelos serviços de nuvem do Microsoft 365 são conhecidos. Além disso, o tráfego e os serviços por si só são protegidos por vários recursos de segurança. Como essa proteção e segurança já está em funcionamento, os dispositivos de borda não precisam duplicá-la. Intermediate destinations and duplicate security processing for Microsoft 365 traffic can dramatically decrease performance.</span></span>

<span data-ttu-id="52743-p103">A primeira etapa para eliminar destinos intermediários e processamento de segurança duplicado é identificar o tráfego do Microsoft 365. - <span data-ttu-id="52743-114">**Otimizar** - Necessário para a conectividade a todos os serviços do Microsoft 365 e representam mais de 75% da largura de banda, conexões e volume de dados do Microsoft 365.</span><span class="sxs-lookup"><span data-stu-id="52743-114">**Optimize** - Required for connectivity to every Microsoft 365 service and represent over 75% of Microsoft 365 bandwidth, connections, and volume of data.</span></span> <span data-ttu-id="52743-115">Esses pontos de extremidade representam os cenários do Microsoft 365 mais confidenciais para o desempenho, latência e disponibilidade da rede.</span><span class="sxs-lookup"><span data-stu-id="52743-115">These endpoints represent Microsoft 365 scenarios that are the most sensitive to network performance, latency, and availability.</span></span>
- <span data-ttu-id="52743-116">**Permitir** - necessário para a conectividade com os recursos e serviços específicos do Microsoft 365, mas não são tão sensíveis ao desempenho e à latência da rede quanto os da categoria Otimizar.</span><span class="sxs-lookup"><span data-stu-id="52743-116">**Allow** - Required for connectivity to specific Microsoft 365 services and features but are not as sensitive to network performance and latency as those in the Optimize category.</span></span>
 - <span data-ttu-id="52743-117">**Padrão** - representam os serviços e dependências do Microsoft 365 que não precisam de otimização.</span><span class="sxs-lookup"><span data-stu-id="52743-117">**Default** - Represent Microsoft 365 services and dependencies that do not require any optimization.</span></span> <span data-ttu-id="52743-118">Você pode considerar os pontos de extremidade de categoria padrão como tráfego normal da Internet.</span><span class="sxs-lookup"><span data-stu-id="52743-118">You can treat Default category endpoints as normal Internet traffic.</span></span>

<span data-ttu-id="52743-119">Você pode encontrar os intervalos de endereços IP e os nomes de domínio DNS em [https://aka.ms/o365endpoints](https://aka.ms/o365endpoints).</span><span class="sxs-lookup"><span data-stu-id="52743-119">You can find the DNS domain names and IP address ranges at [https://aka.ms/o365endpoints](https://aka.ms/o365endpoints).</span></span>

<span data-ttu-id="52743-120">A Microsoft recomenda que você:</span><span class="sxs-lookup"><span data-stu-id="52743-120">Microsoft recommends that you:</span></span>

- <span data-ttu-id="52743-121">Use scripts de Configuração Automática de Proxy (PAC) nos navegadores da Internet dos computadores no local para executar o bypass dos servidores proxy para os nomes de domínio DNS dos serviços baseados na nuvem do Microsoft 365.</span><span class="sxs-lookup"><span data-stu-id="52743-121">Use Proxy Automatic Configuration (PAC) scripts on the Internet browsers of your on-premises computers to bypass your proxy servers for the DNS domain names of Microsoft 365 cloud-based services.</span></span> <span data-ttu-id="52743-122">Para obter o script PAC mais recente do Microsoft 365, confira [Script Get-Pacfile do PowerShellt](https://docs.microsoft.com/office365/enterprise/managing-office-365-endpoints#use-a-pac-file-for-direct-routing-of-vital-office-365-traffic).</span><span class="sxs-lookup"><span data-stu-id="52743-122">For the latest Microsoft 365 PAC script, see the [Get-Pacfile PowerShell script](https://docs.microsoft.com/office365/enterprise/managing-office-365-endpoints#use-a-pac-file-for-direct-routing-of-vital-office-365-traffic).</span></span>

- <span data-ttu-id="52743-p107">Analise seus dispositivos de borda para determinar o processamento duplicado e configure-os para encaminhar o tráfego sem processamento para os pontos de extremidade Otimizar e Permitir, o que é conhecido como executar o bypass de tráfego.</span><span class="sxs-lookup"><span data-stu-id="52743-p107">Analyze your edge devices to determine the duplicate processing and then configure them to forward traffic to Optimize and Allow endpoints without processing. <span data-ttu-id="52743-125">Estas são as recomendações da sua infraestrutura de rede.</span><span class="sxs-lookup"><span data-stu-id="52743-125">Here are these recommendations in your network infrastructure.</span></span>
