# <a name="how-to-manage-microsoft-365-with-windows-powershell-for-delegated-access-permissions-partners"></a>Como gerenciar o Microsoft 365 com Windows PowerShell para parceiros de Permissões de Acesso Delegado

*Esse artigo se aplica ao Microsoft 365 Enterprise e ao Office 365 Enterprise.*

Os Parceiros com Permissão de Acesso Delegada (DAP) são parceiros da Agregação e dos Provedores de Soluções em Nuvem (CSP). Muitos são provedores de rede ou de telecomunicações. Eles agrupam Microsoft 365 assinaturas em suas ofertas de serviço. Quando eles vendem uma assinatura de Microsoft 365, eles são automaticamente concedidos permissões Administer On Behalf Of (AOBO) para as seções do cliente para que possam administrar e relatar essas permissões. Essas tarefas são difíceis de realizar no Microsoft 365 de administração. É muito mais fácil usar o PowerShell para Microsoft 365 tarefas administrativas, como:
- Listar todos os **TenantIds do** cliente e seus domínios 
- Identificar todos os usuários em uma instalação do cliente e suas licenças atribuídas
> [!NOTE]
> Algumas tarefas administrativas só podem ser feitas no PowerShell.
