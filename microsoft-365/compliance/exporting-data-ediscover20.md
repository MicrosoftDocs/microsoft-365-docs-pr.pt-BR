# <a name="export-case-data-in-advanced-ediscovery"></a>Exportar dados de ocorrência na Descoberta eDiscovery Avançada

Há três maneiras de exportar dados de um conjunto de revisão:

**Baixe:** Baixe (usando um navegador) um pequeno conjunto de arquivos nativos. Essa é a maneira mais rápida de exportar um pequeno conjunto de dados. Esse método preserva os nomes de arquivo nativos. Para obter mais informações, [consulte Baixar documentos de um conjunto de revisão.](download-documents-from-review-set.md)

**Exportar:** Personalizar quais dados são exportados. Isso inclui a exportação de metadados de arquivo, arquivos nativos, arquivos de texto e documentos redacionados que foram salvos em um arquivo PDF. Depois que os dados exportados são preparados, você pode baixar os dados da guia **Exportar** para um computador local. Para obter mais informações, [consulte Exportar documentos de um conjunto de revisão](export-documents-from-review-set.md)

**Adicione a outro conjunto de revisão:** Copiar dados de um conjunto de revisão para um conjunto de revisão diferente. Para obter mais informações, [consulte Adicionar dados de um conjunto de revisão para outro conjunto de revisão.](add-data-to-review-set-from-another-review-set.md)

> [!NOTE]
> No Microsoft 365, os dados são hashes e esses hashes são fornecidos no arquivo de saída para fins de verificação. Isso é complementado por logs de auditoria e funcionalidade de relatórios, como estatísticas de coleta, relatórios de conjunto de carga e relatórios de exportação (incluindo o arquivo de carregamento de exportação).
