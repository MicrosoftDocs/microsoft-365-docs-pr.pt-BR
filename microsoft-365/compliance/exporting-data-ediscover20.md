# <a name="export-case-data-in-advanced-ediscovery"></a>Exportar dados de caso em Advanced eDiscovery

Há três maneiras de exportar dados de um conjunto de revisão:

**Baixar:** Baixe (usando um navegador) um pequeno conjunto de arquivos nativos. Essa é a maneira mais rápida de exportar um pequeno conjunto de dados. Esse método preserva os nomes de arquivo nativos. Para obter mais informações, [consulte Download documents from a review set](download-documents-from-review-set.md).

**Exportar:** Personalizar quais dados são exportados. Isso inclui a exportação de metadados de arquivos, arquivos nativos, arquivos de texto e documentos redacted salvos em um arquivo PDF. Depois que os dados exportados são preparados, você pode baixar os dados da guia **Exportar** para um computador local. Para obter mais informações, consulte [Exportar documentos de um conjunto de revisão](export-documents-from-review-set.md)

**Adicione a outro conjunto de revisão:** Copie dados de um conjunto de revisão para um conjunto de revisão diferente. Para obter mais informações, [consulte Add data from one review set to another review set](add-data-to-review-set-from-another-review-set.md).

> [!NOTE]
> Em Microsoft 365, os dados são com hash e esses hashes são fornecidos no arquivo de saída para fins de verificação. Isso é complementado por logs de auditoria e funcionalidade de relatórios, como estatísticas de coleção, relatórios de conjunto de carga e relatórios de exportação (incluindo o arquivo de carga de exportação).
