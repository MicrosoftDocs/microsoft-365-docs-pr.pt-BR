# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>RGPD simplificado: um guia para sua pequena empresa

 *Usando o Microsoft 365 Business para reduzir e gerenciar a conformidade do RGPD* 
  
A regulamentação geral de proteção de dados (RGPD) é uma regulamentação de União Européia (UE) que determina como uma organização deve lidar com dados pessoais. O RGPD deve entrar em vigor em 25 de maio de 2018. Se sua empresa vender o, fornece serviços para ou empregar cidadãos da União Européia, o [rgpd](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) o afetará. 
  
Este artigo ajuda você a entender o que é o RGPD, por que ele vem e como o Microsoft 365 Business pode ajudar sua organização a cumprir o RGPD.
  
## <a name="a-quick-overview-of-the-gdpr"></a>Uma rápida visão geral do RGPD

O RGPD é uma regulamentação da UE que atualiza e expande a diretiva de proteção de dados anterior (DPD) em primeiro lugar no 1995. O RGPD se preocupa com a privacidade dos dados de uma pessoa, seja ele um cliente, cliente, funcionário ou parceiro de negócios. O objetivo do RGPD é reforçar a proteção de dados pessoais para cidadãos da UE, independentemente de residirem na União Européia ou em qualquer outro lugar. A regulamentação define as expectativas e aconselha como alcançá-las. Quando o RGPD se torna aplicável no final de maio de 2018, as organizações devem ter medidas em vigor que atendam aos requisitos do RGPD.
  
O RGPD é sobre dados e como é usado. Pense nos dados como tendo um ciclo de vida. O ciclo é iniciado quando você coleta dados, continua conforme você o armazena e usa (processamento) e termina quando você o exclui completamente dos seus sistemas. 
  
O RGPD é preocupado com os seguintes tipos de dados: 
  
- **Dados pessoais:** Se você puder vincular dados a um indivíduo e identificá-los, esses dados serão considerados pessoais em relação ao RGPD. Exemplos de dados pessoais incluem nome, endereço, data de nascimento e endereço IP. O RGPD considera até mesmo informações codificadas (também conhecidas como informações "pseudônimos") como dados pessoais, independentemente de como obscurecer ou técnicos os dados são, se os dados puderem ser vinculados a um indivíduo.
    
- **Dados pessoais confidenciais** Estes são dados que adicionam mais detalhes a dados pessoais. Os exemplos incluem Religion, associação de União comercial, origem étnica e assim por diante. Dados pessoais confidenciais também incluem dados biométricos e DNA. Em RGPD, os dados confidenciais têm regras de proteção mais rigorosas do que os dados pessoais. 
    
## <a name="gdpr-terms"></a>Termos RGPD

Você verá alguns termos chamados frequentemente no RGPD. É importante entender estes termos.
  
 **Consentimento**
  
O RGPD afirma: "o processamento de dados pessoais deve ser projetado para atender ao Mankind". O RGPD espera atingir essa meta usando o consentimento ao processar dados pessoais. Isso pode ser o ato de perguntar aos seus clientes se eles desejam receber mensagens de email de sua empresa. Isso também significa que não há mais caixas de seleção de recusa no seu site quando você deseja usar dados para marketing. Você deve obter consentimento explícito usando um "Clear affirmative Act". Além disso, você precisará manter registros de quando um consentimento for obtido ou revogado.
  
 **Direitos dos titulares dos dados**
  
O RGPD estabelece direitos de entidades de dados, o que significa que, com relação aos seus dados pessoais, clientes, funcionários, parceiros de negócios, clientes, contratados, estudantes, fornecedores e assim por diante têm o direito de:
  
- **Ser informado sobre seus dados:** Você deve informar as pessoas sobre o uso de seus dados. 
    
- **Ter acesso aos seus dados:** Você deve fornecer aos usuários acesso a qualquer um dos seus dados que você mantém (por exemplo, usando acesso à conta ou de alguma maneira manual). 
    
- **Solicitar a retificação de dados:** As pessoas podem solicitar que você corrija os dados imprecisos. 
    
- **Peça para que os dados sejam excluídos:** Também conhecido como o ' direito de apagamento ', esse direito permite que um indivíduo solicite que qualquer um de seus dados pessoais que uma empresa tenha coletado seja excluído em todos os sistemas que o utilizam ou o compartilhe. 
    
- **Solicitar processamento restrito:** Uma pessoa pode pedir que você suprima ou restrinja seus dados. No entanto, ele só é aplicável em determinadas circunstâncias. 
    
- **Ter portabilidade de dados:** Uma pessoa pode solicitar a transferência dos dados para outra empresa. 
    
- **Objeto:** Um objeto individual pode ter os dados que estão sendo usados para vários usos, incluindo marketing direto. 
    
- **Peça para não estar sujeita à tomada de decisões automatizada, incluindo a criação de perfil:** O RGPD tem regras rígidas sobre o uso de dados para o perfil de pessoas e automatiza decisões com base nessa criação de perfil. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>Perguntas comuns de pequenas empresas sobre o RGPD

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>Sou um proprietário exclusivo eu realmente tenho que me preocupar com o RGPD?

O RGPD é sobre os dados que você processa, e não o número de funcionários que você tem. Ele afeta empresas de todos os tamanhos, mesmo dos próprios proprietários. No entanto, as empresas com menos de 250 funcionários têm algumas isenções, como a redução reduzida de registros, mas somente se você tiver certeza de que o processamento de dados não afeta os direitos da pessoa e é processamento ocasional.
  
Por exemplo, o processamento de dados não pessoais seria isento ou necessidade de medidas reduzidas. No entanto, se você processar quaisquer dados que sejam vistos como "dados confidenciais de categoria especial", mesmo que ocasionalmente, você precisará registrar esse processamento de dados. A definição de "processamento ocasional" é vaga, mas ela se aplica aos dados que são usados uma vez ou raramente.
  
Você também deve certificar-se de que os dados pessoais coletados estão protegidos. Isso significa que você precisa criptografá-lo e certificar-se de que o acesso a ele é controlado usando pelo menos uma senha. Manter os dados do cliente em uma planilha em sua área de trabalho sem proteção não atende às expectativas do RGPD. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>Como saber se o site da nossa empresa é compatível com o RGPD?

A primeira pergunta a ser feita é: você coleta dados pessoais em qualquer lugar do seu site? Por exemplo, você pode ter um formulário de contato que solicita um nome e um endereço de email. Se você deseja enviar emails de marketing, certifique-se de adicionar uma caixa de seleção ' aceitar ' que explica exatamente o que você usará os dados para. Somente se o destinatário verificar essa caixa, você poderá usar seus dados pessoais para fins de marketing.
  
Além disso, verifique se o banco de dados que armazena os dados está protegido. A empresa de hospedagem na Web ou o fornecedor de armazenamento em nuvem poderá avisar sobre isso. Se você usa o Microsoft 365 Business, o armazenamento de dados é compatível com RGPD. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>Minha empresa está fora da Europa. A RGPD realmente nos afeta?

O RGPD é uma regulamentação que protege cidadãos da UE. Se sua empresa lida com o cidadãos da UE agora ou você espera no futuro, você será afetado. Isso se aplica a ambos os cidadãos em todo o estado da UE e os outros.
  
Considere estes exemplos:
  
- Uma empresa norte-americana que contrata carros para o cidadãos da UE precisará satisfazer os requisitos do RGPD ao coletar e processar os dados do cliente. A empresa será obrigada a ter o consentimento quando usar os dados do cliente e garantir que eles sejam armazenados com segurança. Eles também precisarão garantir que o cliente possa aplicar todos os seus direitos de assunto de dados. 
    
- Uma empresa australiana vende produtos online e seus usuários configuram contas online. Os direitos e o consentimento dos dados do RGPD serão aplicados ao cidadãos da UE que abrirem uma conta. A empresa precisará certificar-se de que o cliente pode aplicar todos os seus direitos de assunto de dados. 
    
- Uma caridade internacional coleta dados sobre doadores e os usa para enviar atualizações e solicitações de doações. O RGPD afirma: '... o processamento de dados pessoais para fins de marketing direto pode ser considerado como realizado para um interesse legítimo. " No entanto, a responsabilidade está na organização para comprovar seus interesses que substituem os do assunto dos dados. A empresa (ou neste caso, a organização beneficente) deve sempre ter um consentimento informado, explícito e opcional.
    
O RGPD também se aplica se os dados do cliente forem movidos entre bordas. Se você usar a computação em nuvem para armazenamento de dados, será necessário garantir que o serviço seja totalmente compatível com RGPD. Pode ser complicado se o armazenamento de dados estiver em locais que tenham um registro ruim de proteção de dados. Se você usa o Microsoft 365 Business, temos a documentação legal correta em vigor para cobrir os requisitos do RGPD.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>Certamente, coletar dados, mas outra empresa o armazena. Isso me deixa de sair do gancho?

No RGPD, se você coletar dados, será afetado em alguma extensão. O RGPD tem o conceito de um processador de dados e um controlador de dados: 
  
- **Controlador de dados:** Uma pessoa ou organização (você pode ter controladores conjuntos) que decide como, o que e por que os dados são coletados. Eles podem armazená-lo usando os servidores de nuvem de outros da empresa. Por exemplo, um site que coleta dados do cliente é um controlador. 
    
- **Processador de dados:** Uma pessoa ou organização que armazena dados em nome do (s) controlador (es) e processa esses dados mediante solicitação. Por exemplo, o armazenamento de dados corporativos do Office 365 atua como um processador e é totalmente compatível com RGPD. 
    
    Uma organização ou sistema pode atuar como um controlador e um processador. O Microsoft 365 Business pode atuar como tanto e em conformidade com o RGPD.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>Ainda posso enviar emails de marketing para meus clientes antigos?

Você precisa certificar-se de que seus clientes, mesmo aqueles que você tinha anos, consentiam usar seus dados para marketing. Você pode ter capturado anteriormente o consentimento, bem como um registro para mostrá-lo. Em caso afirmativo, você está pronto para continuar o marketing. Caso contrário, você precisará obter permissão do cliente para continuar o marketing para eles. Isso geralmente envolve o envio de um email solicitando que os clientes vá para seu site e selecionem uma opção para receber emails futuros. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>Tenho que me preocupar com o RGPD ao recrutar novos funcionários? E os funcionários atuais?

O RGPD não afeta apenas os dados do cliente; Ele também se estende aos dados dos funcionários. Os novos recrutadores geralmente estão localizados usando plataformas de mídia social como o LinkedIn. Certifique-se de que você não armazene nenhum dado de recrutamento em potencial sem sua permissão expressa.
  
Como para os funcionários existentes e novos contratos de funcionários, uma assinatura no final de um contrato não assume necessariamente o consentimento, especialmente quando uma cláusula não-affirmative é usada em um contrato. Nesse caso, você deve capturar o consentimento de uma maneira explícita associada à cláusula. O que isso significa depende do seu contrato de funcionário, mas você pode usar "interesse legítimo" em alguns casos e adicionar um aviso de processamento de dados do funcionário para garantir que seus funcionários estejam cientes do que você fará com seus dados.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-business"></a>Satisfazer as preocupações de privacidade usando o Microsoft 365 Business

Tornar-se compatível com o RGPD é garantir que os dados pessoais estejam protegidos. O RGPD tem um conceito conhecido como privacidade por design e padrão. Isso significa que a proteção de dados deve ser "Baked" para um sistema e um produto, de forma que satisfazer as preocupações de privacidade é segunda natureza. 
  
Da mesma forma que seus maiores, uma pequena empresa precisa de sacrificar a segurança. O Microsoft 365 Business foi projetado para empresas de menos de 300 funcionários. Pequenas empresas podem usar as ferramentas baseadas em nuvem da Microsoft para melhorar a produtividade dos negócios. Com o Microsoft 365 Business, uma pequena empresa pode gerenciar emails, documentação e até reuniões e eventos. Ele também tem medidas de segurança internas e gerenciamento de dispositivos, que são essenciais para a conformidade do RGPD.
  
O Microsoft 365 Business pode ajudá-lo com o processo RGPD das seguintes maneiras:
  
- **Detectar:** Uma etapa importante para a conformidade do RGPD é saber quais são os dados que você tem. 
    
- **Gerenciar:** O controle de acesso aos dados e o gerenciamento de seu uso é uma parte integral do RGPD. O Microsoft 365 Business protege dados corporativos com base em políticas que você deseja aplicar aos dispositivos. O gerenciamento de dispositivos é vital em uma idade onde os funcionários trabalham remotamente. O Microsoft 365 Business inclui recursos de gerenciamento de dispositivos que garantem que os dados estejam protegidos em todos os dispositivos. Por exemplo, você pode especificar que todos os dispositivos Windows 10 em sua empresa estejam protegidos através do Windows Defender. 
    
- **Proteger:** O Microsoft 365 Business foi projetado para segurança. Seus controles de gerenciamento de dispositivos e proteção de dados funcionam em sua rede de negócios, incluindo dispositivos remotos, para ajudar a manter os dados seguros. O Microsoft 365 Business oferece controles como configurações de privacidade em aplicativos do Office e criptografia de documentos. Com o Microsoft 365 Business, é possível realizar o monitoramento de conformidade do RGPD para garantir que você tenha o nível certo de proteção. 
    
- **Relatório:** O RGPD coloca bastante ênfase nos relatórios. Mesmo uma empresa com um único funcionário, se essa empresa processa grandes quantidades de dados, é necessário para documentar e relatar seus procedimentos. O Microsoft 365 Business retira a dor de problemas para os requisitos de relatórios de organizações menores. 
    
    Ferramentas como logs de auditoria permitem que você controle e informe a movimentação de dados. Os relatórios incluem classificar os dados que você coleta e armazena, o que você faz com os dados e as transferências dos dados. 
    
Clientes, funcionários e clientes estão se tornando mais cientes da importância da privacidade dos dados e, agora, esperam que uma empresa ou organização respeite essa privacidade. O Microsoft 365 Business fornece as ferramentas para alcançar e manter a conformidade do RGPD sem um upheaval maciço para sua empresa.
