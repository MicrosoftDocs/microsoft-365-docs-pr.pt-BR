# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>RGPD simplificado: um guia para sua pequena empresa

 *Usar o Microsoft 365 para empresas para mitigar e gerenciar a conformidade com o RGPD* 
  
O RGPD (Regulamento Geral de Proteção de Dados) é uma regulamentação da União Europeia (UE) que determina como uma organização deve lidar com dados pessoais. O RGPD deve entrar em vigor em 25 de maio de 2018. Se sua empresa vender para, fornece serviços ou emprega cidadãos da União Europeia, o [RGPD](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) afetará você. 
  
Este artigo ajuda você a entender o que é o RGPD, por que ele veio e como o Microsoft 365 para empresas pode ajudar sua organização a cumprir o RGPD.
  
## <a name="a-quick-overview-of-the-gdpr"></a>Uma visão geral rápida do RGPD

O RGPD é uma regulamentação da UE que atualiza e expande a DPD (Diretiva de Proteção de Dados) anterior, aprovada pela primeira vez em 1995. O RGPD está preocupado com a privacidade dos dados de um indivíduo, seja um cliente, cliente, funcionário ou parceiro comercial individual. O objetivo do RGPD é fortalecer a proteção de dados pessoais para cidadãos da UE, quer eles residam na UE ou em outro lugar. A regulamentação define as expectativas e aconselha como conquistá-las. Quando o RGPD se torna impositiva no final de maio de 2018, as organizações devem ter medidas em execução que atendam aos requisitos do RGPD.
  
O RGPD tem tudo a ver com dados e como eles são usados. Pense nos dados como tendo um ciclo de vida. O ciclo começa quando você coleta dados, continua à medida que você os armazena e os usa (processamento) e termina quando você os exclui completamente de seus sistemas. 
  
O RGPD está preocupado com os seguintes tipos de dados: 
  
- **Dados pessoais:** Se você puder vincular dados a um indivíduo e identificá-los, esses dados serão considerados pessoais em relação ao RGPD. Exemplos de dados pessoais incluem nome, endereço, data de nascimento e endereço IP. O RGPD considera que as informações codificadas (também conhecidas como informações "pseudonimous" sejam dados pessoais, independentemente de quão obscurecidos ou técnicos os dados sejam, se os dados puderem ser vinculados a um indivíduo.
    
- **Dados pessoais confidenciais** São dados que adicionam mais detalhes aos dados pessoais. Exemplos incluem a religião, a associação sindical, a origem étnica e assim por diante. Dados pessoais confidenciais também incluem dados biométricos e DNA. Em RGPD, os dados confidenciais têm regras de proteção mais rigorosas do que dados pessoais. 
    
## <a name="gdpr-terms"></a>Termos RGPD

Você verá alguns termos referidos com frequência no RGPD. É importante entender esses termos.
  
 **Consentimento**
  
O RGPD afirma: "O processamento de dados pessoais deve ser projetado para servir à raça humana". O RGPD espera alcançar essa meta usando o consentimento ao processar dados pessoais. Esse pode ser o simples ato de perguntar aos seus clientes se eles querem receber mensagens de email da sua empresa. Isso também significa que não há mais caixas de seleção de não aceitação em seu site quando você deseja usar dados para marketing. Você deve aceitar o consentimento explícito usando um "ato afirmativo claro". E, você também precisará manter registros de quando um consentimento é tomado ou revogado.
  
 **Direitos das entidades de dados**
  
O RGPD estabelece direitos de assunto de dados, o que significa que, com relação aos seus dados pessoais, clientes, funcionários, parceiros de negócios, clientes, prestadores de serviços, alunos, fornecedores e assim por diante têm o direito de:
  
- **Ser informado sobre seus dados:** Você deve informar as pessoas sobre o uso de seus dados. 
    
- **Ter acesso aos seus dados:** Você deve dar às pessoas acesso a qualquer um de seus dados que você ressarça (por exemplo, usando o acesso à conta ou de alguma maneira manual). 
    
- **Peça retificação de dados:** Os indivíduos podem solicitar que você corrija dados imprecisos. 
    
- **Peça que os dados sejam excluídos:** Também conhecido como "direito de apagamento", esse direito permite que um indivíduo solicite que qualquer um de seus dados pessoais coletados por uma empresa seja excluído em todos os sistemas que os usam ou compartilham. 
    
- **Solicitar processamento restrito:** Um indivíduo pode solicitar que você suprime ou restrinja seus dados. No entanto, ela só é aplicável em determinadas circunstâncias. 
    
- **Ter portabilidade de dados:** Um indivíduo pode solicitar que seus dados sejam transferidos para outra empresa. 
    
- **Objeto:** Um indivíduo pode se opor aos dados que estão sendo usados para vários usos, incluindo marketing direto. 
    
- **Peça para não estar sujeito a decisões automatizadas, incluindo a criação de perfil:** O RGPD tem regras estritas sobre o uso de dados para perfilar pessoas e automatizar decisões com base nessa criação de perfil. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>Perguntas comuns sobre o RGPD

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>Sou um proprietário exclusivo- preciso realmente me preocupar com o RGPD?

O RGPD é sobre os dados que você processa, não o número de funcionários que você tem. Afeta empresas de todos os tamanhos, até mesmo proprietários exclusivos. No entanto, empresas com menos de 250 funcionários têm algumas isenções, como a manutenção de registros reduzidos, mas somente se você tiver certeza de que o processamento de dados não afeta os direitos do indivíduo e é um processamento ocasional.
  
Como exemplo, o processamento de dados não pessoais seria isento ou precisaria de medidas reduzidas. No entanto, se você processar quaisquer dados que são vistos como "dados confidenciais de categoria especial", mesmo que apenas ocasionalmente, você terá que registrar esse processamento de dados. A definição de "processamento ocasional" é vagas, mas deve ser aplicada a dados usados uma vez ou raramente.
  
Você também deve garantir que os dados pessoais coletados estão protegidos. Isso significa que você precisa criptografá-lo e garantir que o acesso a ele seja controlado usando pelo menos uma senha. Manter os dados do cliente em uma planilha em sua área de trabalho sem proteção não atenderá às expectativas do RGPD. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>Como posso saber se nosso site da empresa está em conformidade com o RGPD?

A primeira pergunta a se fazer é: Você coleta dados pessoais em qualquer lugar do seu site? Por exemplo, você pode ter um formulário de contato que solicita um nome e um endereço de email. Se você quiser enviar emails de marketing, adicione uma caixa de seleção de "aceitação" que explica exatamente para o que você usará os dados. Somente se o destinatário verificar essa caixa poderá usar seus dados pessoais para fins de marketing.
  
Além disso, verifique se o banco de dados que armazena os dados está protegido. Sua empresa de hospedagem da Web ou fornecedor de armazenamento em nuvem poderá aconselhá-lo. Se você usar o Microsoft 365 para empresas, o armazenamento de dados será compatível com o RGPD. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>Minha empresa está fora da Europa. O RGPD realmente nos afeta?

O RGPD é uma regulamentação que protege os cidadãos da UE. Se sua empresa lida com cidadãos da UE agora, ou você espera no futuro, será afetado. Isso se aplica aos cidadãos que vivem em um Estado da UE e aos que vivem em outro lugar.
  
Considere estes exemplos:
  
- Uma empresa dos EUA que contrata carros para cidadãos da UE precisará atender aos requisitos do RGPD quando coletar e processar os dados do cliente. A empresa será obrigada a aceitar o consentimento quando eles aceitarem os dados do cliente e garantirem que os dados sejam armazenados com segurança. Eles também precisarão garantir que o cliente possa aplicar todos os seus direitos de assunto de dados. 
    
- Uma empresa australiana vende produtos online e seus usuários configuram contas online. Os direitos e o consentimento dos dados do RGPD serão aplicados aos cidadãos da UE que abrirem uma conta. A empresa precisará garantir que o cliente possa aplicar todos os seus direitos de assunto de dados. 
    
- Uma instituição de caridade internacional coleta dados sobre doadores e os usa para enviar atualizações e solicitações de doações. O RGPD afirma: '... o processamento de dados pessoais para fins de marketing direto pode ser considerado como realizado por um interesse legítimo." No entanto, a responsabilidade é da organização para provar que seus interesses substituem os do assunto de dados. A empresa (ou, nesse caso, a organização de caridosa) deve sempre obter consentimento de aceitação, explícito, explícito.
    
O RGPD também se aplica se os dados do cliente se movem entre bordas. Se você usar a computação em nuvem para armazenamento de dados, precisará garantir que o serviço seja totalmente compatível com o RGPD. Pode ficar complicado se o armazenamento de dados estiver em locais com um registro ruim de proteção de dados. Se você usar o Microsoft 365 para empresas, temos a documentação legal correta para abranger os requisitos do RGPD.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>Claro, eu coleciono dados, mas alguma outra empresa os armazena. Isso me deixa fora do gancho?

No RGPD, se você coletar dados, será afetado até certo ponto. O RGPD tem o conceito de um processador de dados e um controlador de dados: 
  
- **Controlador de Dados:** Uma organização ou indivíduo (você pode ter controladores conjuntos) que decide como, o que e por que os dados são coletados. Eles podem armazená-lo usando servidores de nuvem de outra empresa. Por exemplo, um site que coleta dados do cliente é um controlador. 
    
- **Processador de Dados:** Uma pessoa ou organização que armazena dados em nome dos controladores e processa esses dados mediante solicitação. Por exemplo, o Armazenamento de dados do Microsoft 365 Apps para empresas atua como um processador e é totalmente compatível com o RGPD. 
    
    Uma organização ou sistema pode atuar como um controlador e um processador. O Microsoft 365 para empresas pode atuar como ambos e está em conformidade com o RGPD.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>Ainda posso enviar emails de marketing para meus clientes antigos?

Você precisa garantir que seus clientes, mesmo aqueles que você teve por anos, tenham consentido em usar seus dados para marketing. Você pode ter capturado o consentimento anteriormente, bem como um registro para exibi-lo. Em caso afirmado, você está definido para continuar o marketing. Caso não seja, você precisa obter permissão do cliente para continuar o marketing para eles. Isso geralmente envolve o envio de um email solicitando que os clientes acessem seu site e selecione uma opção para consentir para receber emails futuros. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>Preciso me preocupar com o RGPD quando recrutei novos funcionários? E os funcionários atuais?

O RGPD não afeta apenas os dados do cliente; também se estende aos dados dos funcionários. Os novos recrutas geralmente estão localizados usando plataformas de mídia social, como o LinkedIn. Certifique-se de que você não armazene dados de recrutas potenciais sem a permissão expressa deles.
  
Quanto aos funcionários existentes e novos contratos de funcionários, uma assinatura no final de um contrato não assume necessariamente o consentimento, especialmente quando uma cláusula não afirmativa é usada em um contrato. Nesse caso, você deve capturar o consentimento de forma explícita associada à cláusula. O que isso significa depende do contrato de funcionário, mas você pode usar "interesse legítimo" em alguns casos e adicionar um aviso de processamento de dados de funcionários para garantir que seus funcionários estão cientes do que você fará com seus dados.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>Satisfazer preocupações de privacidade usando o Microsoft 365 para empresas

Tornar-se compatível com o RGPD é garantir que os dados pessoais estão protegidos. O RGPD tem um conceito conhecido como Privacidade por Design e Padrão. Isso significa que a proteção de dados deve ser "cozida" para um sistema e um produto para que satisfazer preocupações de privacidade seja de segunda natureza. 
  
Como suas contrapartes maiores, uma pequena empresa precisa de conveniência sem sacrificar a segurança. O Microsoft 365 para empresas foi projetado para empresas com menos de 300 funcionários. Pequenas empresas podem usar ferramentas baseadas em nuvem da Microsoft para melhorar a produtividade dos negócios. Com o Microsoft 365 para empresas, uma pequena empresa pode gerenciar emails, documentação e até mesmo reuniões e eventos. Ele também tem medidas de segurança internas e gerenciamento de dispositivos, que são vitais para a conformidade do RGPD.
  
O Microsoft 365 para empresas pode ajudá-lo com o processo RGPD das seguintes maneiras:
  
- **Descubra:** Uma etapa importante para a conformidade do RGPD é saber quais dados você tem. 
    
- **Gerenciar:** Controlar o acesso a dados e gerenciar seu uso é uma parte integral do RGPD. O Microsoft 365 para empresas protege os dados comerciais com base nas políticas que você deseja aplicar a dispositivos. O gerenciamento de dispositivos é vital em uma era em que os funcionários trabalham remotamente. O Microsoft 365 para empresas inclui recursos de gerenciamento de dispositivos que garantem que os dados estão protegidos em todos os dispositivos. Por exemplo, você pode especificar que todos os dispositivos Windows 10 em sua empresa são protegidos por Windows Defender. 
    
- **Proteger:** O Microsoft 365 para empresas foi projetado para segurança. Seus controles de gerenciamento de dispositivos e proteção de dados funcionam em toda a rede empresarial, incluindo dispositivos remotos, para ajudar a manter os dados seguros. O Microsoft 365 para empresas oferece controles como configurações de privacidade em aplicativos do Office e criptografia de documentos. Com o Microsoft 365 para empresas, você pode executar o monitoramento de conformidade do RGPD para garantir que você tenha o nível de proteção correto definido. 
    
- **Relatório:** O RGPD dá muita ênfase ao relatório. Mesmo uma empresa com um único funcionário, se essa empresa processa grandes quantidades de dados, é necessária para documentar e relatar seus procedimentos. O Microsoft 365 para empresas tira a dor de cabeça dos requisitos de relatórios para organizações menores. 
    
    Ferramentas como logs de auditoria permitem rastrear e relatar movimentação de dados. Os relatórios incluem a classificação dos dados coletados e do armazenamento, o que você faz com os dados e as transferências dos dados. 
    
Clientes, funcionários e clientes estão se tornando mais cientes da importância da privacidade de dados e agora esperam que uma empresa ou organização respeite essa privacidade. O Microsoft 365 para empresas fornece as ferramentas para alcançar e manter a conformidade do RGPD sem uma grande reavaliação para sua empresa.
