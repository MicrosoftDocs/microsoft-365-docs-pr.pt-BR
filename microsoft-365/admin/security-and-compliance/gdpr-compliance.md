# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>RGPD simplificado: um guia para sua pequena empresa

 *Usar o Microsoft 365 para empresas para reduzir e gerenciar a conformidade com o RGPD* 
  
O RGPD (Regulamento Geral sobre a Proteção de Dados) é uma regulamentação da União Europeia (UE) que determina como uma organização deve lidar com dados pessoais. O RGPD deve entrar em vigor em 25 de maio de 2018. Se sua empresa vender, fornece serviços ou emprega cidadãos da União Europeia, o [RGPD](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) afetará você. 
  
Este artigo ajuda você a entender o que é o RGPD, por que ele veio e como o Microsoft 365 para empresas pode ajudar sua organização a estar em conformidade com o RGPD.
  
## <a name="a-quick-overview-of-the-gdpr"></a>Uma visão geral rápida do RGPD

O RGPD é uma regulamentação da UE que atualiza e expande a DpD (Diretiva de Proteção de Dados) anterior, aplicada pela primeira vez em 1995. O RGPD se preocupa com a privacidade dos dados de um indivíduo, seja um cliente, cliente, funcionário ou parceiro comercial. O objetivo do RGPD é reforçar a proteção de dados pessoais para cidadãos da UE, independentemente de eles residiem na UE ou em outro lugar. A regulamentação define as expectativas e orienta sobre como elas são atingidas. Quando o RGPD se torna aplicação no final de maio de 2018, as organizações devem ter medidas em execução que atendam aos requisitos do RGPD.
  
O RGPD é tudo sobre dados e como eles são usados. Pense nos dados como tendo um ciclo de vida. O ciclo começa quando você coleta dados, continua à medida que você os armazena e usa (processamento) e termina quando você os exclui completamente de seus sistemas. 
  
O RGPD se preocupa com os seguintes tipos de dados: 
  
- **Dados pessoais:** Se você puder vincular dados a um indivíduo e identificá-los, esses dados serão considerados pessoais em relação ao RGPD. Exemplos de dados pessoais incluem nome, endereço, data de nascimento e endereço IP. O RGPD considera até mesmo informações codificadas (também conhecidas como informações "pseudônimos" ) como dados pessoais, independentemente do quanto os dados são ocultos ou técnicos, se os dados puderem ser vinculados a um indivíduo.
    
- **Dados pessoais confidenciais** Esses são os dados que adicionam mais detalhes aos dados pessoais. Exemplos incluem conflitos, associação de união, origem étnica e assim por diante. Dados pessoais confidenciais também incluem dados biométricos e DOLS. De acordo com o RGPD, os dados confidenciais têm regras de proteção mais rigorosas do que os dados pessoais. 
    
## <a name="gdpr-terms"></a>Termos RGPD

Você verá alguns termos referidos com frequência no RGPD. É importante entender esses termos.
  
 **Consentimento**
  
O RGPD declara: "O processamento de dados pessoais deve ser projetado para servir à polícia". O RGPD se permite atingir essa meta usando o consentimento ao processar dados pessoais. Esse pode ser o simples ato de perguntar aos seus clientes se eles querem receber mensagens de email da sua empresa. Isso também significa que não haverá mais caixas de seleção de opção em seu site quando você quiser usar dados para marketing. Você deve consentir explicitamente usando um "ato afirmativo claro". Além disso, você também precisará manter registros de quando um consentimento é feito ou revogado.
  
 **Direitos do assunto dos dados**
  
O RGPD estabelece direitos de assunto de dados, o que significa que, com relação aos seus dados pessoais, clientes, funcionários, parceiros de negócios, clientes, prestadores de serviços, alunos, fornecedores e assim por diante têm o direito de:
  
- **Ser informado sobre seus dados:** Você deve informar as pessoas sobre o uso de seus dados. 
    
- **Ter acesso aos seus dados:** Você deve dar aos indivíduos acesso a qualquer um de seus dados que você mantém (por exemplo, usando o acesso à conta ou de alguma maneira manual). 
    
- **Solicitar retificação de dados:** As pessoas podem solicitar que você corrija dados imprecisos. 
    
- **Peça para os dados serem excluídos:** Também conhecido como "direito de apagar", esse direito permite que um indivíduo solicite que qualquer um dos seus dados pessoais coletados por uma empresa seja excluído em todos os sistemas que os utilizam ou compartilhem. 
    
- **Solicitação de processamento restrito:** Um indivíduo pode solicitar que você suprime ou restrinja seus dados. No entanto, isso só é aplicável em determinadas circunstâncias. 
    
- **Ter portabilidade de dados:** Um indivíduo pode solicitar que seus dados sejam transferidos para outra empresa. 
    
- **Objeto:** Um indivíduo pode se opor aos seus dados que estão sendo usados para vários usos, incluindo marketing direto. 
    
- **Peça para não estar sujeito à tomada de decisões automatizada, incluindo criação de perfil:** O RGPD tem regras estritas sobre como usar dados para perfil de pessoas e automatizar decisões com base nessa criação de perfil. 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>Perguntas comuns de pequenas empresas sobre o RGPD

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>Sou um único proprietário- realmente preciso me preocupar com o RGPD?

O RGPD é sobre os dados que você processa, não o número de funcionários que você tem. Isso afeta empresas de todos os tamanhos, até mesmo proprietários exclusivos. No entanto, as empresas com menos de 250 funcionários têm algumas isenções, como a manutenção de registros reduzidos, mas somente se você tiver certeza de que o processamento de dados não afeta os direitos do indivíduo e é um processamento ocasional.
  
Por exemplo, o processamento de dados não pessoais seria isento ou precisaria de medidas reduzidas. No entanto, se você processar quaisquer dados que são vistos como "dados confidenciais de categoria especial", mesmo que apenas ocasionalmente, você terá que registrar esse processamento de dados. A definição de "processamento ocasional" é vagas, mas deve ser aplicada aos dados usados uma vez ou raramente.
  
Você também deve garantir que os dados pessoais coletados estão protegidos. Isso significa que você precisa criptografá-lo e garantir que o acesso a ele seja controlado usando pelo menos uma senha. Manter os dados do cliente em uma planilha na área de trabalho sem proteção não atenderá às expectativas do RGPD. 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>Como posso saber se o site da empresa é compatível com o RGPD?

A primeira pergunta a se fazer é: Você coleta dados pessoais em qualquer lugar do seu site? Por exemplo, você pode ter um formulário de contato que solicita um nome e um endereço de email. Se você quiser enviar emails de marketing, certifique-se de adicionar uma caixa de seleção de "aceitação" que explica exatamente para que você usará os dados. Somente se o destinatário verificar essa caixa, você poderá usar seus dados pessoais para fins de marketing.
  
Além disso, verifique se o banco de dados que armazena os dados está protegido. Sua empresa de hospedagem na Web ou o fornecedor de armazenamento em nuvem poderão aconselhá-lo. Se você usar o Microsoft 365 para empresas, o armazenamento de dados será compatível com o RGPD. 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>Minha empresa está fora da Europa. O RGPD realmente nos afeta?

O GDPR é uma regulamentação que protege cidadãos da UE. Se sua empresa lida com cidadãos da UE agora ou você espera no futuro, você será afetado. Isso se aplica a cidadãos que estão em um estado da UE e aos que estão em outro lugar.
  
Considere estes exemplos:
  
- Uma empresa dos EUA que contrata carros para cidadãos da UE precisará atender aos requisitos de RGPD quando coletarem e processarem os dados do cliente. A empresa será obrigada a consentir quando receber os dados do cliente e garantir que os dados sejam armazenados com segurança. Eles também precisarão garantir que o cliente possa aplicar todos os seus direitos de assunto de dados. 
    
- Uma empresa australiana vende produtos online e seus usuários configuram contas online. Os direitos e o consentimento do assunto de dados RGPD serão aplicados a cidadãos da UE que abrirem uma conta. A empresa precisará garantir que o cliente possa aplicar todos os seus direitos de assunto de dados. 
    
- Uma instituição internacional coleta dados sobre doadores e os usa para enviar atualizações e solicitações de doações. O RGPD afirma: '... o processamento de dados pessoais para fins de marketing direto pode ser considerado como executado para um interesse legítimo." No entanto, a responsabilidade é da organização provar que seus interesses substituem os do assunto dos dados. A empresa (ou, nesse caso, a organização organizada) deve sempre obter o consentimento informado, explícito e de aceitação.
    
O RGPD também se aplica se os dados do cliente se movem entre bordas. Se você usar a computação em nuvem para armazenamento de dados, precisará garantir que o serviço seja totalmente compatível com o RGPD. Pode ser complicado se o armazenamento de dados estiver em locais com um registro ruim de proteção de dados. Se você usar o Microsoft 365 para empresas, temos a documentação legal correta para abranger os requisitos do RGPD.
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>Claro, eu coleciono dados, mas alguma outra empresa os armazena. Isso me sai do gancho?

De acordo com o RGPD, se você coletar dados, será afetado de alguma forma. O RGPD tem o conceito de processador de dados e controlador de dados: 
  
- **Controlador de dados:** Uma pessoa ou organização (você pode ter controladores conjuntos) que decide como, o que e por que os dados são coletados. Eles podem armazená-lo usando os servidores de nuvem de outra empresa. Por exemplo, um site que coleta dados do cliente é um controlador. 
    
- **Processador de Dados:** Uma pessoa ou organização que armazena dados em nome dos controladores e processa esses dados mediante solicitação. Por exemplo, o armazenamento de dados do Microsoft 365 Apps para empresas atua como um processador e é totalmente compatível com o RGPD. 
    
    Uma organização ou sistema pode atuar como um controlador e um processador. O Microsoft 365 para empresas pode atuar como ambos e está em conformidade com o RGPD.
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>Ainda posso enviar emails de marketing para meus clientes antigos?

Você precisa garantir que seus clientes, mesmo aqueles que você tinha há anos, consentiram em usar seus dados para marketing. Você pode ter capturado anteriormente o consentimento, bem como um registro para exibi-lo. Se sim, você está pronto para continuar o marketing. Caso não seja, você precisa obter permissão do cliente para continuar a comercializar para ele. Isso geralmente envolve o envio de um email solicitando que os clientes acessem seu site e selecione uma opção para consentir em receber emails futuros. 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>Preciso me preocupar com o RGPD quando recrutar novos funcionários? E os funcionários atuais?

O RGPD não afeta apenas os dados do cliente; também se estende aos dados dos funcionários. Os novos recrutadores geralmente estão localizados usando plataformas de mídia social, como o LinkedIn. Certifique-se de não armazenar dados de recrutar em potencial sem a permissão expressa deles.
  
Para funcionários existentes e contratos de novos funcionários, uma assinatura no final de um contrato não assume necessariamente o consentimento, especialmente quando uma cláusula não afirmativa é usada em um contrato. Nesse caso, você deve capturar o consentimento de maneira explícita associada à cláusula. O que isso significa depende do contrato do funcionário, mas você pode usar "interesse legítimo" em alguns casos e adicionar um aviso de processamento de dados do funcionário para garantir que seus funcionários estão cientes do que você fará com os dados deles.
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>Satisfazer preocupações de privacidade usando o Microsoft 365 para empresas

Tornar-se compatível com o RGPD é garantir que os dados pessoais estão protegidos. O RGPD tem um conceito conhecido como Privacidade por Padrão e Design. Isso significa que a proteção de dados deve ser "configurada" em um sistema e em um produto para que a satisfação das preocupações com a privacidade seja de segunda natureza. 
  
Assim como seus equivalentes maiores, uma pequena empresa precisa de conveniência sem comprometer a segurança. O Microsoft 365 para empresas foi projetado para empresas com menos de 300 funcionários. As pequenas empresas podem usar ferramentas baseadas em nuvem da Microsoft para melhorar a produtividade dos negócios. Com o Microsoft 365 para empresas, uma pequena empresa pode gerenciar emails, documentação e até mesmo reuniões e eventos. Ele também tem medidas de segurança internas e gerenciamento de dispositivos, que são vitais para a conformidade com o RGPD.
  
O Microsoft 365 para empresas pode ajudá-lo com o processo de RGPD das seguintes maneiras:
  
- **Descubra:** Uma etapa importante para a conformidade com o RGPD é saber quais dados você tem. 
    
- **Gerencie:** Controlar o acesso aos dados e gerenciar seu uso é uma parte integral do RGPD. O Microsoft 365 para empresas protege os dados comerciais com base nas políticas que você deseja aplicar aos dispositivos. O gerenciamento de dispositivos é essencial em uma idade em que os funcionários trabalham remotamente. O Microsoft 365 para empresas inclui recursos de gerenciamento de dispositivos que garantem que os dados estão protegidos em todos os dispositivos. Por exemplo, você pode especificar que todos os dispositivos Windows 10 em sua empresa estão protegidos por meio do Windows Defender. 
    
- **Proteger:** O Microsoft 365 para empresas foi projetado para segurança. Os controles de proteção de dados e gerenciamento de dispositivos funcionam em toda a rede comercial, incluindo dispositivos remotos, para ajudar a manter os dados seguros. O Microsoft 365 para empresas oferece controles como configurações de privacidade em aplicativos do Office e criptografia de documentos. Com o Microsoft 365 para empresas, você pode executar o monitoramento de conformidade com o RGPD para garantir que tenha o nível de proteção correto definido. 
    
- **Relatório:** O RGPD dá muita ênfase aos relatórios. Mesmo uma empresa com um único funcionário, se essa empresa processa grandes quantidades de dados, é necessária para documentar e relatar seus procedimentos. O Microsoft 365 para empresas tira o trabalho dos requisitos de relatório para organizações menores. 
    
    Ferramentas como logs de auditoria permitem rastrear e relatar a movimentação de dados. Os relatórios incluem a classificação dos dados que você coleta e armazena, o que você faz com os dados e as transferências dos dados. 
    
Clientes, funcionários e clientes estão se tornando mais cientes da importância da privacidade de dados e agora esperam que uma empresa ou organização respeite essa privacidade. O Microsoft 365 para empresas fornece as ferramentas para alcançar e manter a conformidade com o RGPD sem uma grande atividade para sua empresa.
