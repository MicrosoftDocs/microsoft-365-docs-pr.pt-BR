## <a name="structured-versus-unstructured-and-semi-structured-content"></a><span data-ttu-id="a2ac7-114">Conteúdo estruturado versus conteúdo não estruturado e semiestruturado</span><span class="sxs-lookup"><span data-stu-id="a2ac7-114">Structured versus unstructured and semi-structured content</span></span>

<span data-ttu-id="a2ac7-115">Utilize modelos de compreensão de documento para identificar e extrair dados de documentos não estruturados, como cartas ou contratos, onde as entidades de texto que você deseja extrair estão em frases ou regiões específicas do documento.</span><span class="sxs-lookup"><span data-stu-id="a2ac7-115">Use document understanding models to identify and extract data from unstructured documents, such as letters or contracts, where the text entities you want to extract is in sentences or specific regions of the document.</span></span> <span data-ttu-id="a2ac7-116">Por exemplo, um documento não estruturado pode ser uma carta de renovação de contrato que pode ser escrita de maneiras diferentes.</span><span class="sxs-lookup"><span data-stu-id="a2ac7-116">For example, an unstructured document could be a contract renewal letter that can be written in different ways.</span></span> <span data-ttu-id="a2ac7-117">No entanto, as informações existem de forma consistente no corpo de cada documento de renovação de contrato, como a cadeia de texto *Data de início do serviço de* seguida de uma data real.</span><span class="sxs-lookup"><span data-stu-id="a2ac7-117">However, information exists consistently in the body of each contract renewal document, such as the text string *Service start date of* followed by an actual date.</span></span>
